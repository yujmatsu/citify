## 全期間共通の Drop Point 判断ルール

### 緊急時の優先順位

困った時、以下の順序で守る:

1. **7/10 までに提出すること**(これが最優先、何があっても死守)
2. **コア機能(A 群)が動作すること**
3. **DevOps テーマ要件を満たすこと**(CI/CD、IaC、観測性)
4. **AI エージェントの必然性が伝わること**(マルチエージェント構成)
5. **差別化機能(B 群)が動くこと**(優先度高い順:B-2 比較、B-3 Veo、B-4 Imagen)
6. **拡張機能(C 群)**(余力があれば)

### 状況別の判断

- **2 日連続で同じバグに詰まっている** → そのバグを諦めて、機能をスキップ or 簡略化
- **新機能を入れたら既存機能が壊れた** → 即座に Revert、リスクを取らない
- **明らかに時間がない** → C 群から順に削る、次に B 群の優先度低いものから削る
- **自分の体調が悪い** → 半日休む。徹夜開発は禁止(品質が下がりミスが増えるため)
