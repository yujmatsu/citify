## 1. プロダクトビジョン

**「自分の街、自分の世代の話を、60 秒で。」**

Citify は、若者世代が自分の住む自治体・関心のある自治体で起きている政策・行政の動きを、TikTok 的な縦動画フィードで自然に知れるマルチエージェント AI プロダクトです。

自治体ホームページは「図書館」(整然と並んでいるが自分で探しに行く必要がある)。Citify は「For You フィード」(関心軸でキュレーションされ、向こうから流れてくる)。

## 2. ターゲットユーザー

**プライマリ**:18〜35 歳の都市生活者で、自分の自治体に対する解像度が低い層

**3 つの代表ペルソナ**:

### ペルソナ A:新社会人タイプ(22 歳)
- 大学卒業して上京、世田谷区民になったばかり
- 区議会が何を議論しているか想像もつかない
- でも家賃補助・若者向け制度は気になる
- TikTok/YouTube ショートで情報摂取

### ペルソナ B:離れた実家を気にする層(26 歳)
- 地方出身、東京で就職して数年
- 実家(地方都市)の親の介護・空き家問題が気になり始めた
- 自分の今住む街と実家の両方の情報がほしい
- 帰省時に話題にできる地元情報がほしい

### ペルソナ C:移住・引越し検討層(29 歳)
- リモートワーク前提で郊外移住を検討
- 候補 3 自治体の若者支援・起業支援・子育て支援を比較したい
- 単発の検索では情報が分散しすぎて判断できない

## 3. プロダクトの 4 つの差別化価値

自治体 HP・既存ニュースアプリと違って Citify が提供するのは:

1. **横断**:国会議事録 + 自治体議事録 + プレス + オープンデータ + 公報を 1 フィードに統合
2. **パーソナル化**:ユーザーの年代・関心・自治体プロファイルに応じてキュレーション
3. **翻訳**:役所言葉・法律用語を若者向けの平易な言葉に変換
4. **メディア変換**:60 秒動画 + サムネ + 静止画でテキストの読み込みコストをゼロに

加えて構造的に自治体 HP では絶対に提供できない機能:

5. **比較**:複数自治体の同テーマを並べて差分を見る
6. **発見**:自分から検索しなくても関連する話題が流れてくる
7. **時系列**:議論の流れを追える(議事録だけでは断片的)

## 5. 倫理・コンプライアンス制約(絶対遵守)

> Coding Agent はここに記載された制約を絶対に破ってはいけません。違反するコードは生成禁止です。

### 政治的中立性

- **特定政党・特定候補者を推奨する出力は絶対に生成しない**
- 賛成/反対の意見表明は AI が行わない。事実と論点の提示に留める
- 選挙時期の投票推奨は実装しない(「投票へ行こう」程度の一般情報のみ可)
- 党派性のある言い回しは避ける

### Veo / Imagen の利用制約(Google Generative AI Prohibited Use Policy 準拠)

- **実在の政治家・首長・議員の顔・声・名前を含む映像は絶対に生成しない**
- 政策の概念図、街並み、家族、自然、抽象アイコン等の「シーン」のみ生成可
- 全動画・画像に SynthID 透かしを保持
- 出力時に「AI が説明用に作成した動画です」のラベルを必ず表示
- 原典(議事録 URL、行政ページ)へのリンクを必ず併記

### 著作権・利用規約

- 議事録の全文転載はしない。要約 + 原典リンクの形に統一
- スクレイピング先の robots.txt と利用規約を必ず尊重
- レート制限を実装(同一サーバへの連続アクセスは秒単位で間隔)
- 取得データは内部 RAG として保持し、再配布はしない

### 個人情報

- ユーザーの住所は自治体判定にのみ使用、サーバに保存する場合は最小化
- 「みんなの反応」は集計値のみ表示、個別ユーザーは識別不可

このハッカソンにおける Citify の成功は、以下の順序で定義されます:

1. **7/10 までに動作するデプロイ済みプロダクトを提出する**(必達)
2. **一次審査を通過する**(運営事務局審査)
3. **二次審査を通過する**(外部有識者審査)
4. **特別賞以上を受賞する**(10 万円)
5. **優秀賞以上を受賞する**(30 万円)
6. **最優秀賞を受賞する**(50 万円、最終ピッチ進出)

絞り込みの優先順位は、上位の成功条件を犠牲にしてまで下位を狙わないこと。
