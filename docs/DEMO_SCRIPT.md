# DEMO_SCRIPT.md — 最終ピッチ動画スクリプト

> Findy DevOps × AI Agent Hackathon 2026 の **作品提出用デモ動画** (3 分推奨) と **最終ピッチ (8/19) 用プレゼン** のシナリオ・セリフ・カット割りを定義します。
>
> 動画撮影は Week 6（6/30-7/6）に実施予定。Week 5 までに本ファイルを完成させる。

---

## 0. 制作方針

### 0.1 動画の目的（2種類）

| 動画 | 目的 | 長さ | 視聴者 |
|---|---|---|---|
| **A. 提出用デモ動画** | Proto Pedia / Google Form 提出。一次・二次審査で見られる | 3 分以内推奨 | 審査員（運営事務局・外部有識者） |
| **B. 最終ピッチ動画+発表** | 最終ピッチ会場 (8/19) で生披露 | 5-7 分（ピッチ時間に応じ調整） | 審査員 + 観客 (Google渋谷) |

両者で **同じ素材を活用** し、A は短縮版、B はライブ操作含む拡張版にする。

### 0.2 動画の構成原則 (フクシア式)

1. **冒頭3秒で観客の興味を掴む** (痛みのある問い)
2. **15秒で「これは何か」を理解させる** (一言コンセプト)
3. **30秒-2分で体験デモ** (UIを実際に操作)
4. **2分-2分30秒で技術アピール** (マルチエージェント、DevOps)
5. **最後30秒で社会的意義** (なぜこれが必要か)

### 0.3 撮影・編集ツール

- **画面録画**: OBS Studio / Loom
- **編集**: DaVinci Resolve (無料) または CapCut
- **ナレーション**: 自分の声 (Yuji) + Google Cloud TTS (補助ナレ用)
- **BGM**: YouTube オーディオライブラリの royalty-free
- **字幕**: 必ず日本語字幕を焼き込み（観客の8割が音声OFFで見る）

#### 🎤 自己紹介 (00:00-00:30)

> 「皆さん、こんにちは。
> Citify を開発しました、松本 結司です。
>
> 私は地方出身、都内で IT エンジニアとして働いています。
> 上京して 5 年、いまも世田谷区に住んでいます。
> 一度も、区議会の議論を読んだことがありませんでした。
>

>
> その『知らなかった』を変えるために、Citify を作りました。」

#### 🎤 ライブ操作デモ (中盤)

実機で：
1. 自分のスマホに Citify を表示
2. 観客の中から1人を指名 (or 自分の郵便番号)
3. その自治体の今日の議題を表示
4. 「気になる」をタップ
5. 比較ビューでもう1つの自治体を選択

> 「いま、皆さんの目の前で、リアルタイムで議題が表示されています。

#### 🎤 技術的深掘り (後半)

1機能だけを深く見せる。おすすめは **「Iterative Debate (もしくは比較ビュー)」** の協議ログ可視化。

## 3. 想定質疑応答 (Q&A)

最終ピッチで聞かれそうな質問と回答準備：

### Q1: 「政治的中立性をどう担保していますか？」

> 「3 つの層で担保しています。
> 第一に、プロンプトレベルで『賛成・反対を出すな』『党名は名前だけ、評価は出すな』と明示。
> 第二に、出力後のキーワードフィルタで、禁止語を検出したら再生成。
> 第三に、政治家の顔・声・名前を Veo/Imagen で生成しないリストを保持しています。
> もちろん完璧ではないので、ユーザーから報告フォームも提供します」

### Q2: 「議事録のスクレイピングは合法ですか？」

> 「議事録は公開情報で、引用は著作権法で認められています。
> ただし以下を厳守しています:
> ・全文転載はしない (要約 + 原典URL)
> ・robots.txt を尊重
> ・利用規約があれば遵守
> ・レート制限 (1リクエスト5秒間隔)
> NTT-AT の kaigiroku.net 等は、運営にも事前に確認予定です」

### Q3: 「マネタイズはどうしますか？」

> 「フリーミアム + B2G を想定しています:
> ・無料: 個人ユーザー (For You フィード)
> ・有料: 自治体向け『若者からのリアクション分析ダッシュボード』
> ・自治体は若者の関心を可視化できる、ユーザーは無料で使える、Win-Win です」

### Q4: 「LLM のコストは大丈夫ですか？」

> 「コスト削減策を多層で:
> ・分類・影響度は Flash (Pro の 1/10 のコスト)
> ・Veo は上位3議題のみ生成 (残りは Imagen 静止画)
> ・キャッシュ徹底 (同一プロンプトは再利用)
> 月額試算: 100ユーザー時で 約 $30、1000 ユーザー時で 約 $300」

### Q5: 「他の自治体ニュースアプリと何が違いますか？」

> 「最も近いのは『マニフェスト・スイッチ』ですが、選挙特化です。
> 一般的な自治体ニュースアプリは『情報の集約・配信』止まりで、
> Citify は『**翻訳・パーソナル化・比較**』の3点で差別化しています。
> 特に『若者向け翻訳』と『比較ビュー』は私たちの独自機能です」

### Q6: 「ADK を使った理由は？」

> 「マルチエージェントの A2A 通信が標準化されていること、
> Vertex AI Agent Engine への簡単デプロイ、
> Function Calling のサポートが充実、の 3 点です。
> 今回のように 7 体のエージェントが疎結合で連携する設計には最適でした」

### Q7: 「DevOps テーマへの貢献は？」

> 「Citify そのものが『つくる → まわす → とどける』のフルサイクルを体現しています。
> ・つくる: マルチエージェントによるコンテンツ生成
> ・まわす: 毎朝 5 時の自動収集、評価、改善ループ
> ・とどける: Cloud Run で 1000 ユーザーまでスケール可能
> Terraform で全インフラ管理、GitHub Actions で CI/CD 自動化、
> Cloud Logging + Trace で観測性も担保しています」
